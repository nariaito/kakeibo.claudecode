# カード家計簿

クレジットカードの利用通知メール（Gmail）を自動取得し、カテゴリ別に集計・可視化するスマホ向けWebアプリです。
データはスマホのブラウザに保存されます（Firebaseなし・ログイン不要）。

## Gmail同期の設定

### 1. Google Cloud で Gmail API を有効化

1. https://console.cloud.google.com/ を開く（カード通知が届くGmailアカウントでログイン）
2. プロジェクトを作成または選択
3. 「**APIとサービス**」→「**ライブラリ**」→「Gmail API」→「**有効にする**」
4. 「**APIとサービス**」→「**認証情報**」→「**＋ 認証情報を作成**」→「**OAuth クライアント ID**」
5. アプリの種類：「**ウェブアプリケーション**」
6. 「**承認済みの JavaScript 生成元**」に追加：
   ```
   https://nariaito.github.io
   ```
7. 「**テストユーザー**」に自分のGmailアドレスを追加（OAuth同意画面から設定）
8. 作成 → **クライアントID** をコピー

### 2. アプリに設定

1. アプリの「**設定**」タブを開く
2. 「**Google Client ID**」に貼り付け → 「**保存**」
3. 「**📧 メール同期**」ボタンをタップ
4. Googleアカウント選択画面 → 自分のアカウントを選択
5. 「このアプリはGoogleで確認されていません」→「**詳細**」→「**続行**」

## 仕様・よくある質問

### データはどこに保存される？
スマホのブラウザ（localStorage）に保存されます。サーバーやクラウドには送信されません。

### ブラウザのデータを消すとどうなる？
アプリ本体（GitHub Pages）は消えませんが、入力した明細・予算・設定がリセットされます。
ブラウザの「サイトデータを削除」を実行するときは注意してください。

### 同じメールが2回取り込まれることはある？
ありません。一度取り込んだメールのIDを記録しており、同じメールは2回目以降無視されます。

### 過去のメールも取り込める？
取り込めます。既読・未読は関係なく、件名に以下のキーワードが含まれるメールが対象です：
- ご利用のお知らせ
- カードご利用
- ご利用確認
- 利用確認
- ご利用代金

### 1回の同期で何件まで取得できる？
最大100件です。

### メール同期は自動で行われる？
されません。「📧 メール同期」ボタンを手動で押したときだけ取得します。
カードを使って通知メールが届いたタイミングでボタンを押すのがおすすめです。

### キーワードを変更したら既存の明細は変わる？
変わりません。キーワードは次回のメール同期時にだけ使われます。
既存の明細を新しいキーワードで振り直したい場合は、設定タブの「🔄 既存明細にカテゴリを再適用」ボタンを押してください。

### Client ID を再設定が必要なのはいつ？
以下のときだけです。それ以外は一度設定すれば消えません。
- ブラウザの「サイトデータを削除」をしてしまったとき
- 設定タブの「全データをリセット」をしたとき

## 機能一覧

| 機能 | 内容 |
|------|------|
| **予算管理** | カテゴリ別に月ごとの予算を設定。残り金額を表示 |
| **Gmail同期** | カード通知メールを手動ボタンで取得・解析 |
| **自動振り分け** | 店名キーワードで食費・物品費・その他に自動分類 |
| **その他カテゴリ** | キーワードに一致しない明細は「その他」に集計。ダッシュボードにも表示 |
| **一括再カテゴライズ** | 設定タブの「🔄 既存明細にカテゴリを再適用」で全明細を振り直せる |
| **グラフ** | カテゴリ別ドーナツグラフ ＋ 6ヶ月分の月別棒グラフ |
| **更新ボタン** | ダッシュボードの「🔄 更新」ボタンで最新データに手動更新 |
| **カテゴリ管理** | 設定タブで追加・編集・削除・キーワード設定 |
| **手動追加** | 明細タブから手動で明細を追加・編集・削除 |
